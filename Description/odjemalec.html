Funkcionalnosti aplikacije HandyDandy

1. Brskanje in iskanje vodičev

  • Uporabnik lahko pridobi seznam vseh vodičev.
  • Možnost filtriranja vodičev po kategorijah.
  • Podrobni ogled posameznega vodiča s slikami, videi in seznamom potrebnih orodij.
  • REST API: GET /api/tutorials, /api/tutorials/:id
  • Vhod: /
  • Izhod: seznam vodičev.

2. Dodajanje novih vodičev

  • Registrirani uporabniki lahko dodajo nove vodiče s podrobnostmi, slikami in videoposnetki.
  • Možnost kategorizacije vodičev.
  • REST API: POST /api/tutorials
  • Vhod: JSON objekt vodiča.
  • Izhod: ID novo dodanega vodiča.

3. Brisanje vodičev

  • Registrirani uporabniki lahko izbrišejo svoje vodiče.
  • REST API: DELETE /api/tutorials/:id
  • Vhod: ID vodiča
  • Izhod: potrditev o uspešnem brisanju.

4. Dodajanje in pregledovanje komentarjev

  • Uporabniki lahko dodajajo komentarje k vodičem.
  • Prikaz vseh komentarjev za določen vodič.
  • REST API: GET /api/comments/:tutorialId, POST /api/comments
  • Vhod: JSON objekt komentarja.
  • Izhod: ID novega komentarja.

5. Pregled orodij/materialov

  • Seznam vseh priporočenih orodij in materialov.
  • Podrobnosti o posameznem orodju, vključno z opisom in priporočili za nakup.
  • REST API: GET /api/tools, GET /api/tools/:id
  • Vhod: /
  • Izhod: seznam orodij/materialov.

6. Shranjevanje vodičev za kasnejšo uporabo

  • Uporabniki lahko shranijo svoje priljubljene vodiče za kasnejši ogled.
  • Shranjen vodiči so dostopni tudi brez povezave.
  • REST API: GET /api/user/:id/saved-tutorials
  • Vhod: ID uporabnika.
  • Izhod: seznam shranjenih vodičev.

7. Beleženje napredka

  • Uporabniki lahko beležijo napredek popravila.
  • Popravila so razdelejna po korakih. Uporabnik si lahko beleži, na katerem koraku je ostal.
  • Vhod: JSON objekt napredka (ID vodiča, korak).
  • Izhod: potrditev napredka.

8. Zbiranje značk za uspešno opravljena popravila

  • Uporabnikom je omogočeno zbiranje značk.
  • Ko uporabnik zaključi s popravilom, dobi značko, ki se mu shrani na njegov profil.
  • Vhod: ID opravljenega popravila.
  • Izhod: ID nove značke.

9. Pregled priporočil za nakup materialov in orodij

  • Po pregledu vsega materiala in orodij je uporabnikom predlagano, kje lahko te stvari nabavijo.
  • Uporabnikom so tako olajšana popravila, saj jim aplikacija predlaga, kje lahko dobi potreben material.
  • Vhod: seznam potrebnih orodij/materialov.
  • Izhod: seznam priporočil za nakup (povezave do trgovin).
